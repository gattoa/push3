# Daily Workout — `/workout`

> Primary screen. What users see every time they open the app. Lives within the Workout tab of the persistent two-tab bottom nav.

## Role

This is the app's home screen post-onboarding. It shows today's prescribed workout and lets users log sets immediately. The goal is zero taps between opening the app and seeing what to do. Contextual banners (plan review, check-in) appear above the content area when triggered by the weekly cycle.

## Design Decisions

### Header
- **Left:** List icon — navigates to weekly agenda (`/plan`). This is a secondary view within the Workout tab, not a separate navigation destination.
- **Center:** Day + date (e.g., "Monday, Mar 20"), workout/split name below (e.g., "Push Day")
- **Right:** Avatar with sign-out dropdown
- No week number — it's an internal concept, not user-facing
- No back button — this is home

### Banner System
- A single banner slot sits above the progress bar. At most one banner at a time.
- **Plan review banner:** Appears after new plan generation. Tapping navigates to `/plan` for full review and editing.
- **Check-in banner:** Appears when check-in day arrives (Sunday for POC). Tapping navigates to `/check-in`.
- **Persistence:** Both banners persist 48 hours or until dismissed, whichever comes first.
- See [UX-DESIGN-BRIEF.md](UX-DESIGN-BRIEF.md) Section 4 (Banner System) for full specification.

### Today Detection
- Map current day of week (Mon=0 … Sun=6) to the matching `planned_day.day_index`
- If today is a rest day: show rest day state (see States section)

### Progress Bar
- Same pattern as existing workout view: completed + skipped sets / total sets
- Accent green fill, smooth transition on update

### Exercise Cards
- Carry forward existing design from `/workout/[day]`
- Numbered badge, exercise name (display font), optional notes
- **Historical reference:** "Last: [weight]×[reps]" line below exercise name when historical data exists. Omitted on Week 1 (cold start — no data).
- **PR icon:** Appears on a set row when the logged set's estimated 1RM (Epley: weight × (1 + reps/30)) exceeds the user's previous best for that exercise.
- **No pre-filled inputs:** The AI's prescribed target weight is the primary historical intelligence. Pre-filling with last session data would conflict with progressive overload recommendations.
- Set rows with target, weight input, reps input, complete/skip actions

### Exercise Swaps
- **Trigger:** Swipe left on an exercise card to initiate a swap.
- **Expansion:** Card expands inline (below exercise header, above set table) showing 3 alternative exercises as compact rows (name + equipment tag). Original exercise remains visible at top for comparison.
- **Action:** Tapping an alternative replaces the exercise in place — sets reset, no logged data carries over. Cancel collapses the expansion.
- **Data source (target):** Alternatives are pre-generated by the AI during weekly plan generation. Stored in `planned_exercises.alternatives` (JSON array of exercise IDs). Fetched with the plan — zero latency at swap time.
- **Fallback:** ExerciseDB query (same target muscle + user's equipment − current day's exercises) if pre-generated alternatives are unavailable.
- **Implementation flag:** Requires isolated prompt testing before implementation. See [UX-DESIGN-BRIEF.md](UX-DESIGN-BRIEF.md) Decision 5.

### Set Logging
- Identical behavior to current `/workout/[day]`: immediate save via `/api/log-set`
- Complete (✓) and skip (✗) buttons, mutually exclusive
- Inputs disabled when set is skipped

### Completion Summary
- Appears when all sets have a status (completed or skipped)
- Shows: completed count, skipped count, total volume
- Single action: "View Weekly Plan" (→ `/plan`). Check-in is not embedded here — it surfaces via the banner system when the check-in day arrives.

## States

| State | Description |
|---|---|
| **Fresh** | No sets logged today. All inputs empty, no status icons active. |
| **In Progress** | Some sets logged. Progress bar partially filled. |
| **Complete** | All sets have status. Summary card visible. |
| **Rest Day** | Today has no exercises. Shows: (1) week-so-far summary (days trained, sets completed, completion %), and (2) next training day preview with full exercise list — read-only, muted styling, labeled "Up Next — [Day]." Exercise names link to `/exercise/[id]`. |
| **Rest Day (Week 1 Cold Start)** | No workouts logged yet. Week-so-far summary replaced with a motivational welcome message (e.g. "Your first week — focus on learning the movements and finding your weights"). |
| **Cold Start (Week 1)** | No "Last" reference line. No target weights (by design — no baseline exists). Exercise cards show name, notes, and empty inputs only. |
| **No Plan** | User has no active plan. Redirect to generation or show empty state. |
